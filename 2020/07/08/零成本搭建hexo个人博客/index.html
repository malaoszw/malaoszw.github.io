零成本搭建hexo个人博客

发表于2020-07-08字数统计1.1k阅读时长6分阅读次数

相信每个程序员都有自己想要的个人网站，而且最好是不花一分钱的那种，越方便越好。今天我们就讲一下hexo这个精致、小巧、高逼格的静态博客，还有怎么部署到gitee(码云)、github上。

  • github提供开启公共库（Public）的 Pages 是免费的，私有要收费，没有其他巴拉巴拉的条件； gitee私有仓库同样可以免费使用 Pages 服务，但是需要账号绑定手机号码；coding私有仓库同样可以免费使用 Pages 服务，提供独立模块，但是是最麻烦的，限制组内人数，还要提供身份证、手机号码，所以不考虑coding。

1. 新建仓库

  • github
    首先 github 新建一个仓库，仓库名必须为 <username>.github.io 格式，其中 username 是你github 的昵称。因为github是自动配置的，如果不这么命名，Pages将无法配置。
  • gitee
    gitee 新建一个仓库，方法差不多，仓库名可以随便命名，为了避免那些不必要的麻烦，还是命名为 <username> 格式，其中 `` 是你`gitee 的空间地址。如果忘记的，可以点击设置找到个人空间地址查看一下

2. 安装前提所需相关工具

https://hexo.io/zh-cn/docs/

  • 需要先安装node.js(百度搜一下就知道怎么安装了,这里就不讲了)不然无法使用npm软件包管理工具，Node Package(包) Manager(管理器)。需要安装git，同样是百度一下怎么安装使用（建议先git操作尝试推送成功后再来）。
  • 如果没有vscode，建议还是下载个vscode来操作一下比较方便，对与代码的操作，这里就不讲vscode怎么下载安装了，运行vscode的新终端，cd到自己想要的路径下面(不懂cd是啥的，百度一下)，接下来我们就正式开始。

3. 全局安装hexo

我们可以参考hexo的官方文档来操作 https://hexo.io/zh-cn/

全局安装hexo（如果安装很慢，先去安装cnpm，百度一下怎么安装，安装完成后可以替代 npm为cnpm）

等待执行完毕后，执行下一条命令，初始化项目
hexo init blog中的blog是可以自己随便命名的，执行命令后将会把项目克隆到这个文件夹下面

接着进入你命名的这个文件夹

接着我们在这个目录下安装

然后我们编译启动hexo server可以省略为hexo s

如图编译成功后，我们就可以访问http://localhost:4000/

然后可以修改网站的语言，把en改成zh-CN就可以把网站改为中文

因为改的是整个网站的语言，你需要重新清一遍已编译的静态网页，重新生成
现在终端Ctrl+C让进程中止，然后执行命令hexo clean清除已编译静态网页

执行hexo generate生成静态网页，hexo generate同样可以省略为hexo g

你可以执行下列命令来创建一篇新文章或者新的页面。他会在source/_posts生成mardown文件，

如果还想了解其他主题的可以参考一下官方文档https://hexo.io/themes/

4. 部署

在项目根目录下找到 _congif.yml，找到 deploy

用户名和空间地址自己改，这里就不再说了。

接着在终端上执行hexo deploy省略为hexo d

将会同时推送到远程仓库

  • github
    执行完毕后，github会自动部署，直接访问https://用户名.github.io/

  • gitee
    gitee则需要手动执行，选择服务里的Gitee Pages

    勾选 https后启动

    部署完毕后
    会给你一个地址 https://空间地址.gitee.io
    然后就可以访问了。
